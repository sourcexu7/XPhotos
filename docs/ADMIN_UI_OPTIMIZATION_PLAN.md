# 后台 UI 全局优化计划与进度

## 目标

- 统一后台页面结构与视觉语言，降低认知负担
- 修复明显的导航与信息错误（死链、写死用户信息）
- 提升高频页面可用性（上传、图片列表、设置）
- 建立可追踪的改造记录，支持分批验收

## 范围

- 路由：`/admin/**`
- 组件：`components/admin/**`、`components/layout/admin/**`
- 不包含：业务接口协议调整与数据库结构变更

## 验收标准（当前轮）

- 后台主页面有一致的标题/描述信息区
- 不存在明显无效导航入口
- 侧边栏显示真实用户数据（无会话时回退默认值）
- 上传模式切换清晰、可见、可快速切换
- 图片列表筛选行为对用户可解释
